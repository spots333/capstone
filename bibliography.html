Backtesting Theory

These resources provided the conceptual foundation for what backtesting is, how it is used in practice, and what its limitations are when applied to historical data. They informed how results were interpreted. The resources also inspired many of the trading strategies I included in the final project.

Backtesting in Trading: Definition, Benefits, and Limitations. (n.d.). Investopedia. Retrieved March 5, 2026, from https://www.investopedia.com/terms/b/backtesting.asp
Backtesting Trading Strategies | Complete Guide. (n.d.). Avatrade. Retrieved March 5, 2026, from https://www.avatrade.com/education/online-trading-strategies/backtesting-trading-strategies

Figma

Figma was used to prototype and plan the layout of the user-facing components. This resource helped me learn core UI/UX principles, particularly around making the interface intuitive for users who are not familiar with prediction markets or trading data.

Design Basics: UI/UX, Prototyping & Core Principles. (n.d.). Figma. Retrieved March 5, 2026, from https://www.figma.com/resource-library/design-basics/

JSON

JSON was the primary data format returned by the Kalshi API, so understanding its structure and syntax was necessary. More importantly, JSON is the method used to store market data in the project. It provides the backbone for the entire project and beat SQLite in terms of speed and efficiency.

JSON. (n.d.). Retrieved March 5, 2026, from https://www.json.org/json-en.html

Kalshi API

The Kalshi API was the core data source for this project. The documentation was referenced a lot when getting historical market candlestick data and setting up authentication. Additionally, historical candlestick data was only available to authenticated API requests which meant fetching market/series data on the go was impossible.

Get Historical Market Candlesticks. (n.d.). API Documentation. Retrieved March 5, 2026, from https://docs.kalshi.com/api-reference/historical/get-historical-market-candlesticks
Quick Start: Authenticated Requests (No SDK). (n.d.). API Documentation. Retrieved March 5, 2026, from https://docs.kalshi.com/getting_started/quick_start_authenticated_requests

SQLite

SQLite was tested to store and query the market data collected from the Kalshi API. Upon testing, SQLite was found to not be effective in lowering the storage size of market data. More importantly, fetching data from the database took about 40% longer than from JSON files, killing its possible implementation in the project.

Appropriate Uses For SQLite. (n.d.). Retrieved March 5, 2026, from https://www.sqlite.org/whentouse.html
